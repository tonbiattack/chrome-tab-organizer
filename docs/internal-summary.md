# Project Summary

このドキュメントは、この拡張の現在の仕様と設計上の判断をまとめた概要です。

## 今回追加した機能

### 設定保存

`chrome.storage.local` を使って次を保存するようにした。

- `全ウィンドウを対象にする`
- 標準ルールの有効 / 無効
- カスタムルール

## カスタムルール UI

popup からカスタムルールを追加できるようにした。

`Jira課題キー` `Google Doc ID` `URLパターン` を 1 つのルールに任意に組み合わせられる。

カスタムルールで設定できる内容:

- グループ名
- グループ色
- Jira 課題キー、Google Doc ID、URL パターン
- 有効 / 無効

また、現在開いている Jira タブ URL から課題キーを拾って入力欄へ流し込む補助ボタンと、Google ドキュメント URL から Doc ID を拾う補助ボタンを追加した。

カスタムルールは JSON 形式でエクスポート / インポートできる。

## Jira 向けの使い方

想定ユースケース:

- 親課題に紐づく子課題タブをまとめる
- エピックやスプリント単位でタブをまとめる
- 任意の業務単位名で Jira タブグループを作る
- Jira タブと関連 Google ドキュメントを同じグループにまとめる

現状の仕様:

- Jira API 連携はしていない
- 親子関係は自動取得しない
- 課題キー集合をユーザーが定義する

## 優先順位

グルーピング時の優先順位は次の通り。

`カスタムルール > 標準ルール`

そのため、既存の Jira ルールに該当する URL でも、カスタムルールが先に一致すればそちらを優先する。

## 公開についての判断

### 公開価値

公開価値はある。理由は次の通り。

- 単一目的のタブ管理拡張として説明しやすい
- Jira 課題キー単位でまとめられる点に実用上の差別化がある
- 個人利用や社内利用で即効性がある

### そのまま公開してよいか

Chrome Web Store にそのまま即公開する前に、いくつか整備した方がよい点があります。

主な理由:

- プライバシー説明の整備が不足しやすい
- ストア掲載文やスクリーンショットが未整備
- カスタムルール周辺のテストがまだ薄い
- Jira API を使っていない点を誤解なく説明する必要がある

### ローカル利用ならどうか

`git clone` や ZIP 配布で使うだけなら、Chrome Web Store 向け資料は必須ではありません。  
ただし、最低限の説明として次は用意しておくことが推奨されます。

- 導入手順
- 権限説明
- 保存内容
- 制約事項
